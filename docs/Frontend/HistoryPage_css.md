# HistoryPage.css

Styles for the user's booking history and travel records.

## Purpose
Supports the UI in `HistoryPage.jsx`, providing a clean, timeline-like view of past trips.

## Key Styles
- **History Card**: A complex card layout summarizing each booking, including guest details, travelers, and start date.
- **Tag Cloud**: Optimized styling for displaying chosen activities and facilities as tags.
- **Status Badge**: Styling for confirmed or pending status indicators.
- **Empty State**: Centered illustration and call-to-action for users with no booking history.
- **Header Section**: Standardized header with a back button and title group.
- **Footer**: Detailed summary area for total paid amount and confirmation badges.
