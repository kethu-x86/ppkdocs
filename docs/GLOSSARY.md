# 📖 Glossary of Terms

A quick guide to the key terms and concepts used in the Personalized Travel Planning System.

---

### 🗺️ District
Refers to the 14 administrative districts of **Kerala** (e.g., Kasaragod, Wayanad, Idukki). In this application, a district is the primary unit for itinerary building.

### 🏩 HID (Hotel ID)
A unique numerical identifier assigned to every hotel in the registry. It is used to link hotels to districts, rooms, and payments in the database.

### 🛏️ RID (Room ID)
A unique identifier for a specific room reservation. Each time a user selects a room type, a new RID is generated and linked to the hotel.

### 💰 Budget Tracking
A real-time system that monitors the user's spending against their pre-defined budget. It includes the base room price, nights, and all selected facilities.

### ✨ AI-Suggested Stays
Hotels that are programmatically recommended to the user when no direct matches are found for their specific activity selections or budget constraints.

### ⛽ Commute Estimation
The calculated travel distance and time (for both cars and bikes) between a hotel and the user's chosen activities in a district.

## 🛠️ Technical Terms

- **Vite**: The modern build tool used for the React frontend.
- **Flask**: The Python web framework used for the backend API.
- **CORS**: Cross-Origin Resource Sharing, a mechanism that allows the frontend to securely communicate with the backend.
- **SPA**: Single-Page Application, a web application that interacts with the user by dynamically rewriting the current page rather than loading entire new pages from a server.
